    <!-- Application Structure Plan: The application is designed as a single-page dashboard. The core idea is to move away from the linear document structure and allow users to immediately access relevant content. The top navigation provides access to the main pillars of the report: 'Fondamentaux' (Basics), 'Départements' (The core prompt library), and 'Déploiement' (Strategic part for managers). The main interaction is centered around the department selector, which dynamically loads department-specific use cases as interactive cards. This approach is chosen for usability, as employees will most likely want to see prompts for their specific roles first. An interactive ROI calculator is included to transform a theoretical concept from the report into an engaging, practical tool. A language switcher has been added to the header. -->

    <!-- Visualization & Content Choices: 

        - Report Info: Department-specific prompt examples (tables in the report). -> Goal: Inform, Organize. -> Viz/Presentation Method: Interactive Cards. -> Interaction: User clicks a department button, JS dynamically generates cards for each prompt. Clicking a card reveals details. -> Justification: Much more engaging and less overwhelming than static tables. Allows users to focus on one use case at a time. -> Library/Method: Vanilla JS DOM Manipulation.

        - Report Info: ROI calculation formula. -> Goal: Engage, Demonstrate Value. -> Viz/Presentation Method: An interactive form linked to a Chart.js bar chart. -> Interaction: User adjusts sliders for 'Heures Gagnées' and 'Coût Horaire', the chart and ROI text update in real-time. -> Justification: Turns a static formula into a personalized tool, making the concept of ROI tangible and impactful. -> Library/Method: Chart.js.

        - Report Info: Structure of a perfect prompt. -> Goal: Inform, Educate. -> Viz/Presentation Method: A static visual diagram built with HTML/CSS. -> Interaction: None. -> Justification: A visual layout of the 4 components (Rôle, Contexte, Tâche, Format) is easier to remember than a simple list. -> Library/Method: Tailwind CSS.

        - Report Info: Key risks of AI usage. -> Goal: Inform, Warn. -> Viz/Presentation Method: An accordion component. -> Interaction: User clicks to expand and read details for each risk. -> Justification: Prevents information overload by keeping detailed text hidden by default. -> Library/Method: Vanilla JS.

        - ✨ Gemini Feature 1: Idea Generator. -> Goal: Extend content dynamically. -> Presentation: Button in each department section. -> Interaction: User clicks, sees a loading state, then new cards with AI-generated ideas appear. -> Justification: Makes the library a living tool, not a static document. -> Library/Method: Gemini API Fetch.

        - ✨ Gemini Feature 2: Prompt Enhancer. -> Goal: Teach by doing. -> Presentation: A new "Workshop" section. -> Interaction: User enters a basic idea, clicks a button, and sees their idea transformed into a well-structured prompt. -> Justification: Provides direct, practical value and reinforces learning. -> Library/Method: Gemini API Fetch.
